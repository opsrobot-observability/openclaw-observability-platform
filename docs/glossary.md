# 统一名词列表 (Glossary)

本文档整理了 OpenClaw 可观测性平台（opsRobot）中广泛使用的核心产品概念、技术术语和业务名词，旨在为用户、开发者以及运维人员提供统一的沟通标准。

| 中文名词 | 英文名词 | 名词解释 |
| :--- | :--- | :--- |
| **智能体** | Agent | 具备感知环境、进行决策、调用工具并执行特定任务的人工智能实体。 |
| **数字员工** | Digital Employee | 将 AI 智能体封装为拟人化角色的业务概念，为企业提供特定场景和岗位的服务实体，通常具有独立的能力与安全画像。 |
| **会话** | Session | 用户与智能体或数字员工之间建立的交互周期，包含一轮或多轮的对话、思考链路及任务执行过程。 |
| **可观测性** | Observability | 通过收集和分析日志、指标、链路等数据，对系统（特别是大模型和 Agent 运行架构）内部执行状态的深度透视与“白盒化”能力。 |
| **链路追踪** | Tracing / Session Tracing | 采用标准化协议记录智能体在执行任务时的完整调用链、时间线、状态流转和上下文信息，以实现故障精准定位及全生命周期保障。 |
| **合规审计** | Audit / Compliance Audit | 记录、重放和检查系统中发生的所有关键操作与模型交互日志，建立控制机制（如越权管控、风暴拦截），确保业务符合安全与合规要求。 |
| **画像** | Profile / Portrait | 针对数字员工等对象多维数据的立体刻画与评估模型，主要包括但不限于能力评估、安全防护指数以及交互效率特征。 |
| **算力成本** | Compute Cost / ROI | 指维持智能体运转所消耗的底层推理运算成本，平台采用多维核算模型将其转化成业务部门或员工维度的费用消耗，拒绝算力“糊涂账”。 |
| **Token 消耗** | Token Consumption | 大模型在处理自然语言输入（Prompt）和生成输出（Completion）过程中消耗的基础文本切片单元，直接关联系统调用成本。 |
| **探针/采集器** | Probe / Collector | 部署在模型服务所在节点上的数据采集组件（如 Vector 或基于 eBPF 技术），用于无侵入或低侵入地抓取并上报运行态数据。 |
| **OpenClaw** | OpenClaw | 核心的 AI 智能体调度和服务管理基座，本项目为其提供底层与上层的监控及运营管理支撑。 |
| **opsRobot** | opsRobot | 本项目的内部核心代号及代码库名称，即聚焦于全天候观测、风险感知与生产力评估的可观测性平台。 |
| **OTel** | OpenTelemetry (OTel) | 业界主流的开源可观测性框架与协议规范，项目利用其进行跨服务、跨 Agent 步骤的打点及全链路数据上报。 |
| **eBPF** | eBPF | 一种 Linux 内核级技术，用于在不修改内核源码与应用源码的前提下，安全和高效地收集底层运行与网络监控数据。 |
