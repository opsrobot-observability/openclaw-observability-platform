# AI 用量 KPI 事件截图取证脚本

本文档用于配合营销文章截图，展示 opsRobot 如何通过“可管理的成本”和“会话链路溯源”识别 AI 用量 KPI 误导、循环调用、Token 异常消耗，并完成复盘取证。

核心会话 ID：`sess_ai_kpi_loop_20260407`

## 截图 1：可管理的成本发现异常

页面：成本分析

截图步骤：

1. 启动 mock 环境：`npm run dev:mock`。
2. 进入“成本分析”页面，时间范围选择近 7 天或近 14 天。
3. 截取顶部 KPI 与趋势图：当日 Token 约 `3.18B`，环比 `+486.7%`。
4. 截取 Agent 占比：`乐园运营助手` 位居第一，约 `3.11B` Token。
5. 截取 Top 会话：`sess_ai_kpi_loop_20260407` 位居榜首，约 `3120M` Token。

营销文案：

opsRobot 不只看“AI 用得多不多”，而是把 Token、模型、Agent、会话、成本中心放到同一个成本治理视图里。异常峰值出现时，管理者能第一时间看到是哪一个数字员工、哪一个模型、哪一个会话推高了成本。

## 截图 2：Agent 成本明细定位责任主体

页面：Agent 成本明细

截图步骤：

1. 从成本分析页点击 `乐园运营助手`，或在 Agent 成本明细页搜索 `乐园运营助手`。
2. 展开该行，截取 TCO、模型路由和异常损耗。
3. 重点保留：`3.11B` Token、`50,000` 次调用、异常 Token `3.08B`、异常占比 `99.0%`、`Claude 3.7 Sonnet` 占比 `99.4%`。
4. 点击“查看会话”，进入会话成本明细。

营销文案：

传统监控可能只看到 API 花费变高，但 opsRobot 可以把异常归因到具体数字员工和模型路由。这里能看到问题不是所有 AI 应用一起上涨，而是一个被 KPI 驱动的运营助手在集中消耗。

## 截图 3：会话成本明细确认循环调用

页面：会话成本明细

截图步骤：

1. 筛选 Agent：`乐园运营助手`。
2. 展开 `sess_ai_kpi_loop_20260407`。
3. 截取诊断面板：状态为循环调用，停止原因为 `cost_guardrail_soft_cap_exceeded + max_iterations`。
4. 保留输入 Token `1.54B`、输出 Token `1.58B`、成本约 `¥9360.0000`、交互步骤 `50000`、持续时间 `21586s`。
5. 点击“查看 JSON”，进入会话审计。

营销文案：

“用量高”本身不是问题，问题是高用量有没有业务产出、有没有预算边界、有没有停止条件。opsRobot 会把循环、截断、超预算、异常中断等状态沉淀为可筛选、可排序、可导出的会话明细。

## 截图 4：会话审计展示原始证据

页面：会话审计

截图步骤：

1. 搜索或打开 `sess_ai_kpi_loop_20260407`。
2. 截取会话基本信息：Agent 为 `乐园运营助手`，来源为 `campaign-console`，总 Token 为 `3,120,000,000`。
3. 截取风险摘要：高危/中危/低危规则命中，工具调用数 `12`。
4. 展开关键日志行：
   - 用户提示词要求“提高 AI 使用次数”。
   - `policy.get_usage_budget` 返回软上限和硬上限。
   - `llm.batch_generate` 返回重复提示率 `0.94` 和 Token 消耗。
   - `metrics.increment_ai_clicks` 显示 KPI 计数被批量写入。
   - `cost.internal` 成本快照显示当日 `3,120,000,000` Token、约 `¥9360`。

营销文案：

这一步回答“为什么发生”：会话审计能看到原始 prompt、模型输出、工具调用、风险规则和成本快照。复盘不再依赖口头描述，而是用同一个 session_id 串起完整证据链。

## 截图 5：全链路溯源形成复盘闭环

页面：全链路溯源

截图步骤：

1. 搜索 `sess_ai_kpi_loop_20260407`。
2. 截取“索引元信息”：标题、渠道、租户、总 Token。
3. 展开时间线节点：创建 KPI 任务、读取预算上限、展开 4,987 名员工批处理、第一轮 11.2 亿 Token、KPI 写入后转化下降、第二轮累计 31.2 亿 Token、成本守卫阻断、证据归档。

营销文案：

opsRobot 的价值不止是“发现贵了”，而是能复盘“贵在哪里、为什么贵、是谁触发、系统做了什么、下一次如何阻断”。成本治理和会话溯源合在一起，才能把 AI 从不可控试验变成可管理的生产系统。

## 文章收束句

当 AI 变成 KPI，企业需要的不是更高的调用次数，而是可解释、可归因、可治理的 AI 使用。opsRobot 通过可管理的成本和会话链路溯源，让每一次异常消耗都能被及时发现、精准定位、完整复盘。
