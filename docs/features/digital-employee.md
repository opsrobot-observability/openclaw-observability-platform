# 数字员工（概览与画像）

数字员工模块用于从“员工视角”观察 AI Agent 的运行质量、成本效率与安全风险，包含 **数字员工概览** 与 **数字员工画像** 两个页面，分别用于全局盘点与单体诊断。

---

## 🌟 功能价值

在多 Agent 并行运行的生产环境下，数字员工模块帮助团队：

1. **统一盘点员工健康度**：快速识别高风险员工、在线员工与关键渠道分布。
2. **建立五维诊断闭环**：围绕能力、质量、效能、成本、安全进行单员工深度分析。
3. **支撑治理与优化动作**：将异常会话、风险波动与成本结构关联到具体 `session_key`，便于回溯与修正。

---

## 📊 页面结构

### 1) 数字员工概览

- **核心指标**：健康度、员工总数、在线员工数、渠道分布、人均成本、人均效率、红线员工数。
- **驾驶舱图表**：
  - 成本趋势（时间窗口）
  - 会话趋势（时间窗口）
  - 能力分布（按技能数）
  - 效能分布（P95 耗时）
  - 质量分布（成功率）
  - 安全分布（红灯数量）
- **Top 排行**：高风险清单 / 高效率清单，支持跳转到画像页。

### 2) 数字员工画像

- **员工列表**：按会话展示，单选高亮，支持渠道/风险/模型供应商筛选。
- **基础信息区**：系统标识、最近活跃、标签、模型与工作区信息。
- **五维页签**：
  - 能力维度
  - 质量维度（支持异常明细弹窗）
  - 效能维度
  - 成本维度（Token 与单价、成本估算）
  - 安全维度（风险评分与等级规则）
- **会话时间线**：按风险等级展示会话节点，并可跳转会话溯源。

---

## 🧮 关键口径说明

- **在线员工数**：最近 15 分钟内有会话更新的员工数。
- **红线员工数**：`securityRiskScore >= 70` 的员工数。
- **总 Token**：`输入 Token + 输出 Token`。
- **成本估算**：`(输入Token/1M)*输入单价 + (输出Token/1M)*输出单价`。

## 🎭 Mock 使用方式

当配置 `VITE_MOCK=true` 时，数字员工接口将走 mock 路由：

- `GET /api/digital-employees/overview`
- `GET /api/digital-employees/profile`

对应数据文件：

- `mock/data/digital-employee-overview.mjs`
- `mock/data/digital-employee-profile.mjs`

这两份 mock 已对齐当前页面所需字段结构（含 `o1_summary`、`agentsAggregated`、`tabs`、`qualityDetails` 等），可用于本地无 Doris 场景下的联调与 UI 验证。
