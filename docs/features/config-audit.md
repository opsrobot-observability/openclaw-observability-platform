# 特权与配置变更审计功能说明

当 AI Agent 拥有对业务系统的实质性调用权限（如 Shell 执行、文件系统读写等 API）时，追踪其对基础环境的变更就成了安全管理的重中之重。配置变更审计模块即用于无遗漏地记录数字员工引发的底层操作。

---

## 一、模块定位

在复杂的大模型外挂工具流时代，配置变更审计主要解决以下问题：
- 错误指令导致环境损毁：AI 生成的代码一旦恶意或误删关键路径，缺乏还原和追踪线索。
- 越权写操作泛滥：无法区分出哪些重要文件是被人工修改的，哪些是被自动数字员工“悄悄”变更的。
- 缺乏事后复盘对比：只知道文件发生过改变，却难以查明修改的具体差异及执行当时的上下文。

该模块通过“强制写监控 + 操作进程锁定 + 配置状态 Diff”实现底层的防篡改及溯源。

---

## 二、功能实现说明

### 1) 变更轨迹清单（列表与看板）
针对数字员工向宿主机请求的文件和配置变更，平台生成的聚合视图：
- **事件高光筛选**：按最近七天提供写操作频次的波形柱状图，支持过滤按修改者或改动路径检索。
- **运行特征剥离**：每个写入事件明确保留如操作类型、应用所在运行目录（Workdir）以及执行具体行使进程 (PID)。

### 2) 网关与穿透等级检测
- **执行边界标识**：系统判断出文件变更发生时所穿透的沙箱或物理环境等级，追踪记录如从隔离节点向主节点反向写等越权特征。

### 3) 文件状态对比机制 (Diff Trace)
对于被修改的重要文件，进一步提供配置还原的线索页：
- **留存前后快照**：识别操作属于文件创建、单纯追加 (append)、直接覆盖还是危险的重命名挪用。
- **变更对比 (Diff) 解析**：遇到 YAML、JSON 类型的关键配置遭到修改时，直接联动显示底层的格式变更核对文本。

---

## 三、业务价值

### 1) 保障基础 IT 架构安全
- 让数字员工拥有强大操作能力的同时，保证他们不能越过安全合规边界“毁坏”环境而无需担责。

### 2) 提供无可辩驳的操作证据链
- 即使机器指令出错导致系统级故障（如清空节点模块或者数据库配置文件），也能秒级找出事端发起者及其请求 payload。

### 3) 加速恢复周期
- 依赖于强大的前后状态快照和 Diff 留存，当系统瘫痪时可极大缩短事故抢修与文件恢复的时间窗口。

---

## 四、典型应用场景

- **核心目录入侵排查**：发现 `/etc/hosts`、授权密钥文件、依赖树库被未授权修改时，立即追溯并封禁出问题的 Agent。
- **环境漂移还原**：当原本健康的应用突然不能启动，通过此面板查证最近 24 小时有哪些由 Agent 下发的非预期配置修改引起了偏离。
- **合规操作验核**：满足内控制度关于自动化工具修改物理节点状态必须具有独立审计记录的公司规范。

---

通过持续留存基础配置变更的监控数据，平台给大模型驱动的全自动执行工具装上了企业级的“刹车和后视镜”系统。
