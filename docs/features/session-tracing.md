# 会话审计与溯源追踪 (Session Tracing)

这个模块是 OpenClaw 平台解决“大模型黑盒困境”真正的重武器。它直连存储在 Apache Doris 中的流水记录，向管理员、测试者或安全小组提供了细粒度的微观洞悉。

---

## 🌟 核心价值

当我们面临一个反馈不佳的数字员工交互时（比如答非所问、或把内部系统搞挂了），传统方式只能大海捞针般去查阅生硬的 API 返回字符串。
通过对任何历史会话（哪怕是处于极长生命周期的持久化后台会话）进行深潜追查，你可以精确重构 Agent 每一步是在「想什么」、又“调用了什么”。

## 🔍 功能细节解析

### 1. 全文检索与高级过滤
海量的交互文本如果没有搜索引擎将会是一摊死水。在会话列表上方，你可以运用：
- **混合条件过滤**：支持指定大模型供应商（如 OpenAI、Anthropic）、会话 ID、触发的用户（Channel / Account ID）进行硬性筛选。
- **时间窗筛选**：锁定那一天夜里的某几点故障期间发生的全部对话。

### 2. 会话卡片详情与流转阶段展示

当选中某条会话列表详情后，系统会展示一场“数字员工与真实世界交互的全景动画图”：
- **从发端开始**：用户通过哪个钉钉或 Slack 渠道下达的 `User Message` 命令被录入。
- **思考脑图 (Thinking Chain)**：捕获 Agent 推理规划的思维链条，弄懂它为何判定了目前的解决步骤。
- **过程调动**：多次与系统内环境（例如 `bash`, `fs` 文件读写）或外部 Web 网络获取的工具链（Tools Call）。
- **终态判定**：到底有没有成功投递正确响应给人类客户（系统结束或者被强行切断/中止）。

### 3. 可视化报错溯源高亮 (Risk Nodes Isolation)

为了协助安全团队快速在一场有着 50 到 100 个交互回话堆叠的长上下文中锁定危机点，平台会：
- 自动通过背景色与节点边框进行**醒目标注**。
- `🔴 高风险`：比如代码返回非 `0` 的执行退出码 (Exit Codes)、尝试越狱读取特权私钥。
- `🟡 中风险` / `🟢 无问题`。
- **原始快照提供**：系统为包含特定 `stdout/stderr` 标准错误的输出片段留存代码块快照，无需登入生产业务机，直接在页面里即可阅读堆栈异常。

---

> **最佳实践 (Best Practice) 👉 及时复盘异常 Token 抛耗**
> 建议定期抽样那些 Token 花费极高却没得到什么实质产出的死循环任务，通过完整流水线回放去排查是否某一个 `Prompt System Template` 在面对特殊用户提问时，陷入了过度调用工具却没有跳出退守机制的逻辑漏洞。
