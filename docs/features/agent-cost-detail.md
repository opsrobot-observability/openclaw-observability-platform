# Agent 成本详情功能说明

数字员工算力消耗并不均衡分布。相较于“平台大锅饭”式的核算体系，Agent 成本详情板块致力于将计算、模型调用与带宽资源消耗向下精算落实到单个确定的角色实体甚至单一工作流中，从而精准锚定产出比黑洞节点。

---

## 一、模块定位

在推行自动化数字职员运营架构时，需要直面如下困境：
- 价值评估模糊：不知道哪个数字岗位的投入产出比最高，无法建立针对性扩招或裁撤的考量基准。
- 用度滥用溯源困难：某些非核心智能辅助模块消耗了团队绝大多数调用资金账单，却没有及早报警拦截。
- 横向对比机制匮乏：不同类型服务模块之间的 Token 利用率、失败率缺乏同等基数下的比照。

“成本定向穿透 + 排名揭示机制 + 能力画像交互联通”是该模块实施精准控制的抓手。

---

## 二、功能实现说明

### 1) 单体数字角色账户拆分报表 (Bill per Agent)
为当前全量的 Agent 按指定周期切割消费细单：
- **消耗结构大公开**：揭示某位具体编号/名称下的职员产生的确切输入 Token 及生成回流消耗比例。
- **总花费与响应计次计算**：明确展现每次工作应答任务对应的平均资源购买开支情况。

### 2) 多对象花销打榜比较图谱
让业务层有手段发现用度大头：
- **消费规模矩阵/气泡榜单**：以直方图等形式，明显划定出极高花销与平缓调用的各类角色的横向对比差距图，一戳便探明“算力刺客”。
- **活跃与成本对消度量**：衡量消耗增长和实际输出工单处理等价值曲线的吻合度差值。

### 3) 跨功能画像纵深下钻联动
- 针对表现突兀且造价昂贵的指定员工，允许操作者顺流跳转至该对象的 **[数字员工概览画像](digital-employee.md)** 做综合质量、意图及安全体检评估。

---

## 三、业务价值

### 1) 支撑阿米巴精确投产核算体系
- 让实际使用业务部认领数字员工产生的账期明细表成为真实可能，破除了 AI 开销一笔糊涂大账的现象。

### 2) 促进提示词流程裁剪的直接动力
- 一旦某岗位的单次花费呈现惊人偏高离谱，立即提醒工程师介入实施提示词（Prompt）工程瘦身术和工具连环压缩重组。

---

## 四、典型应用场景

- **月末虚拟员工财务复核**：给不同职能（如辅助客服 Agent 与 代码审计专家 Agent）分别生成费用支出饼图上交管理高层评价效能。
- **僵尸应用清退清理运动**：发现并强制下线资源占用过大却无合理业务流转产出的边缘冗余角色服务。

---

基于细分视角的独立 Agent 记账册功能，为推动更加健康的 AI 化应用生命周期管理与商业化变结算夯实了地基。
