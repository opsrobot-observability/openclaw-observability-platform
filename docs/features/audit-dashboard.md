# 审计概览看板功能说明

此看板是数字员工（AI Agent）运行状态与安全风险的核心监测中心。它为 IT 运维和安全团队提供全局视图，协助系统性地管理智能化应用的运行现状与风险暴露敞口。

---

## 一、模块定位

在接入大量内部机器人或自动化任务后，审计概览看板主要解决以下问题：
- 并发请求不可见：多 Agent 并发产生海量会话，缺少系统级的心跳与并发负载视图。
- 风险感知滞后：被系统安全策略拦截的操作和潜在异常没有聚合入口，难以防范攻击或故障。
- 难以锁定高频调用源：系统无法直观看出谁在过度使用机器，或者哪些工具最容易报错。

该模块通过“总体量感控 + 风险大盘提取 + 异常热点追踪”实现全局安全态势感知。

---

## 二、功能实现说明

### 1) 多时间窗状态汇编（页面级）
以多种时间周期呈现平台资源的存活及负荷统计：
- **今日 / 本周 / 本月分片对比**：快速比对外放接口及并发压力。
- **独立实体存活数**：实时观测活跃会话数、当前用户请求数及有效设备连接等指标。

### 2) 风险与合规大盘
基于底层拦截网关产生的数据进行安全定级聚合：
- **智能红绿灯机制**：量化统计被诊断为高/中/低风险的请求序列。
- **异常占比换算**：测算风险会话池相对于整体执行的比例，动态把控系统污染程度。

### 3) 可视化图表联动
将孤立安全事件转化为全局趋势走向：
- **操作成份拆分**：安全分类结构饼图。
- **风险爆发趋势**：利用平滑曲线映射各等级风险动作发生的高峰时间窗。
- **压力走势评估**：过去七天的整体请求波段统计。

### 4) 高危实体排行 (Top Insights)
输出用于人工介入排查的处方列表：
- **高频消耗用户**：针对并发及 Token 资源进行排行。
- **高频风险工具**：提取持续抛错或触发系统拒绝操作的后端节点（如 `message_tool_name` 等）。

---

## 三、业务价值

### 1) 提升全局运维视力
- 基于时间抽样和实时大盘，全面掌握资源调用量和流量，告别黑盒排障。

### 2) 保障系统运行底线
- 可直观看出当前风险请求是否超出安全阈值，防止小错误引发整体服务的拖慢或系统级熔断。

### 3) 优化排险路径
- 通过梳理出 “Top 风险实体” 榜单，使得专家排错能够优先处理对业务影响最严重的拦截事件链。

---

## 四、典型应用场景

- **安全防御复盘**：每日或每周巡检风险拦截趋势图，评估新上线的防御策略是否起效。
- **异常资源滥用追测**：利用排行榜排查某单一用户或节点是否在遭遇恶意刷号或遭受脚本密集调用。
- **健康度快照**：CIO 掌握数字员工生态圈的安全健康状态汇报。

---

通过审计仪表板提供的宏观风险画像，管理者能实现从“被动接收报警”向“主动防御治理”的体系升级。
