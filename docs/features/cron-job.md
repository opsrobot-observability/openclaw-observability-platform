# 模块概要介绍

OpenClaw 定时任务可观测模块提供面向运维的OpenClaw定时任务的全面观测，提供任务运行状态全局监控、任务列表管理、执行日志追溯与单任务详情分析能力。整合了任务执行数据、性能指标、失败原因诊断、Token 消耗分析与任务执行链路追溯，支持多时间维度筛选、多条件检索与单任务深度监控，帮助运维管理员实现对所有定时任务的全局掌控、异常快速定位与问题闭环，保障平台定时任务体系的稳定、可靠运行。

## 运行概览（全局驾驶舱）

提供定时任务体系的全局可视化监控视图，通过多维度指标聚合与趋势分析，帮助用户快速掌握整体任务运行健康度。

- 支持按时间维度筛选与多图表联动分析，帮助用户识别任务执行波动、高消耗场景与高频失败原因，为优化资源分配与排查系统性问题提供数据支撑。
- 时间范围筛选：支持「最近 7 天 / 30 天 / 90 天」快速切换，也可自定义开始 / 结束时间，适配不同周期的监控分析需求。
- 核心指标聚合：直观展示任务总数、最近一次执行失败数、存在失败的任务数、总执行次数、成功 / 失败次数与整体成功率。
- 执行趋势分析：按日期展示任务执行的「成功 / 失败」次数分布与平均耗时曲线，识别执行量与性能波动。
- Token 消耗监控：按日期与类型展示 Token 消耗趋势与分布占比，定位高消耗任务，优化资源分配。
- 失败原因诊断：通过环形图直观呈现失败原因分布，聚焦高频问题根源。
- 每日执行概览热力图：以颜色深浅展示每日任务成功率，快速识别异常波动日期。
- 任务 / 作业 Top10 分析：按执行次数、失败次数、耗时等维度排序，识别高负载、低稳定性或性能异常的任务。

## 任务监控列表

提供定时任务的集中式列表监控能力，支持多维度筛选、检索与状态监控，实现对所有定时任务的高效管控。

- 提供任务启停控制与列表视图切换功能，适配临时运维场景与不同使用习惯，保障任务体系的灵活管控。
- 任务概览卡片：展示任务总数、启用任务数、停用任务数，快速感知整体任务状态。
- 多维度筛选检索：支持关键词模糊搜索（任务名称 / ID/Agent/ 执行策略 / 状态），以及启用状态、最近执行结果、所属 Agent 组合筛选。
- 任务列表详情：集中展示任务基础信息（名称、ID、Agent、执行策略）、计划信息（下次执行时间）、执行统计（次数、失败数、成功率、平均 / 最大耗时）与状态信息（最近执行结果、时间、耗时、成功时间、启用状态）。
- 任务启停控制：支持直接查看任务启用状态，可快速切换启停状态，适配临时运维场景。
- 列表视图切换与刷新：支持表格 / 卡片视图切换，一键刷新获取最新任务状态。
- 分页数据管理：支持分页加载任务列表，底部展示数据范围与总条数，适配大规模任务场景。

## 运行日志

提供定时任务所有执行记录的集中式查询与详情查看能力，完整记录每次任务执行的过程与结果，支持问题追溯与审计。

- 支持多条件检索与日志详情展开，帮助用户快速定位单次执行的异常问题，实现故障闭环。
- 时间范围筛选：支持「最近 24 小时 / 7 天 / 30 天 / 90 天」快速切换，也可自定义时间范围。
- 多条件检索：支持按任务 ID、名称、运行 ID、错误信息等关键词搜索，按 Agent、状态筛选。
- 执行记录列表：展示运行 ID、任务 ID、任务名称、Agent、投递状态、执行状态、开始 / 结束时间、耗时、Token 消耗、下次运行时间与错误信息。
- 日志详情查看：支持展开查看单次执行的完整日志（摘要、原始 JSON、输出详情），定位执行过程问题。
- 分页与刷新：支持分页加载日志列表，一键刷新获取最新执行记录。

## 任务详情（单任务深度分析）

### 任务概览

提供单个定时任务的基础信息与最近执行状态的聚合视图，帮助用户快速了解任务的配置信息与运行表现。

- 集中展示任务的配置信息、执行策略与最近执行状态，让用户快速掌握任务的基础情况与健康度。
- 完整呈现任务的执行目标与步骤说明，帮助用户理解任务逻辑，降低运维沟通成本。
- 基础信息展示：任务 ID、名称、描述、启用状态、所属 Agent、会话 Key、唤醒 / 投递方式、创建 / 更新时间。
- 最近执行状态：下次执行时间、最近执行时间 / 状态 / 耗时、最近投递状态、连续错误数与最近错误信息。
- 任务内容详情：完整展示任务的执行目标、步骤、约束条件与期望输出格式，便于理解任务逻辑。

### 运行监控

提供单个定时任务的历史执行数据可视化分析能力，通过多维度指标与图表，帮助用户评估任务稳定性与性能表现。

- 以时间维度聚合展示任务的执行成功率、耗时变化与结果分布，直观呈现任务的稳定性与性能波动，为任务优化提供数据支撑。
- 支持多时间范围筛选，适配短期排查与长期趋势分析需求，帮助用户识别任务执行的周期性问题与性能瓶颈。
- 时间范围筛选：支持多时间维度查看任务执行数据。
- 核心指标卡片：展示总运行次数、成功 / 失败次数与成功率。
- 每日执行概览热力图：按日期展示任务执行次数与成功率，识别执行波动。
- 执行趋势分析：按日期展示执行成功 / 失败次数与平均耗时变化，定位性能异常。
- 结果构成分布：通过环形图展示成功 / 失败占比，直观呈现任务稳定性。

### 运行日志

提供单个定时任务所有执行记录的专属查询与筛选能力，聚焦单任务的执行过程，实现问题的精准追溯。

- 针对单个任务的执行记录进行集中管理，支持多条件筛选与分页查询，帮助用户快速定位特定执行的异常问题。
- 完整留存单任务的所有执行日志，为任务优化、故障排查与合规审计提供专属数据依据。
- 时间范围筛选：支持多时间维度查看单任务执行日志。
- 多条件检索：支持按状态、模型、Agent、会话 ID、错误信息等筛选执行记录。
- 执行记录列表：展示事件时间、运行时间、状态、耗时、投递状态、模型、Agent、会话 ID、下次执行时间与错误信息。
- 分页数据管理：支持分页加载日志列表，适配大量执行记录场景。

### 运行追溯

提供单个定时任务执行过程的全链路追溯能力，通过时间线与单次执行详情，帮助用户定位问题发生的节点与原因。

- 以时间线形式呈现任务的执行结果分布，直观呈现任务执行的波动情况，帮助用户识别异常执行的时间节点。
- 提供单次执行的完整链路信息与追溯入口，支持跳转至 Agent 与会话详情页面，实现从任务到执行链路的全流程排查，提升问题解决效率。
- 执行趋势时间线：按日期展示任务执行结果分布（成功 / 失败 / 运行中 / 其他），直观呈现执行波动。
- 单次执行详情：展示运行时间、状态、耗时、投递方式、Agent、会话 ID、下次执行时间、摘要输出与错误信息。
- 链路追溯入口：支持跳转至 Agent 追溯、会话追溯与运行详情页面，实现执行链路全流程排查。
