# 日志搜索

## 1. 功能简介

日志搜索是 **OpsRobot** 面向运维、研发与安全的 **统一日志检索入口**，对接 Doris 中的多类落库日志，用一套界面完成时间圈定、条件组合与结果下钻，支撑 OpenClaw 场景下的 **线上排障**、**安全合规审计** 与 **问题举证留痕**。

### 1.1 灵活时间筛选

支持 **从秒级预设到自然年等日历粒度** 的预设时间窗（相对时间与自然日/周/月/年等），以及 **自定义起止时间**；时间与其它检索条件调整后需 **手动应用** 再触发查询，避免频繁自动请求，在宽时间窗或大结果集下更有利于 **提升检索效率** 与后端稳定性。

### 1.2 源日志统一检索

覆盖 **Agent 会话**、**配置变更（审计）**、**网关路由** 等多类日志；既支持 **单数据源** 精确查询，也支持 **多源合并** 后按时间归并展示。可结合界面中的 **日志来源 / 类别** 快速收窄参与检索或合并的表范围，减少无关噪声。

### 1.3 高效检索与过滤

支持 **关键词组合检索**（`AND` / `OR` / `NOT`、引号短语、通配符 `*`、`?` 等，具体以后端解析为准）。可与 **Trace ID**、**Session ID**、**Request ID**、**Agent**、**网关模块**、**日志级别** 等 **结构化条件叠加**；在支持的数据源上还可 **仅看错误 / 排除错误**，便于快速收敛异常日志。

### 1.4 友好结果交互

提供 **列显隐控制**、侧栏 **字段取值分布**（基于当前页样本）、单行 **字段视图与原始 JSON 展开**、**时间分布趋势图**（可调整分桶粒度、**点击柱形下钻** 收窄时间范围）、**分页** 与 **结果导出** 等能力，便于在宽表中定位异常、对照原始落库内容并 **留存证据**。



## 2. 功能点（按分类）

以下与「功能简介」中的四个能力维度一一对应（其中 **源日志统一检索** 指多种日志来源在同一检索链路下的统一查询与合并，与界面「数据源 / 全部」能力一致），便于查阅实现细节。

### 2.1 灵活时间筛选

1. **时间预设**：支持从秒级（如最近 30 秒）到自然日/周/月/年等多种相对与日历预设，快速圈定故障窗口。
2. **自定义时间**：自定义起止时间（`datetime-local`），需开始 ≤ 结束；非法范围会提示并阻止请求。
3. **时间应用**：修改时间条件后通过「应用」刷新查询，避免每次键入都触发检索。

### 2.2 源日志统一检索

1. **会话 / Agent 日志**：对应 Doris 中 `agent_sessions_logs`（及按日分表等命名规则下的表），侧重 LLM/Agent 会话、工具调用、消息角色等字段。
2. **配置变更日志**：对应 `audit_logs`，侧重配置与安全相关操作记录。
3. **网关日志**：对应 `gateway_logs`，可配合「模块」等条件过滤路由、HTTP 等子系统。
4. **全部（多源合并）**：在服务端将多表结果按时间归并排序，统一展示；可与「日志来源（筛选）」联动，决定参与合并的表集合。
5. **日志来源（类别）**：如 Agent、Gateway、Skill、Channel 等，用于在「全部」模式下收窄参与合并的数据源子集（与产品内映射说明一致）。

### 2.3 高效检索与过滤

1. **全文检索与条件组合**：支持 **AND / OR / NOT**、**引号短语**、通配符 `*`、`?` 等（具体语法以后端解析为准）；可与会话、Trace、Request、Agent、Channel、级别等 **结构化条件同时生效**，便于「先缩窗再全文」或「先全文再缩列」。
2. **会话与链路标识**：**Session ID** 精确匹配；**Trace ID**、**Request ID** 从 `log_attributes` 中解析（兼容多种 JSON 键名），支撑链路追踪。
3. **Agent、模型与通道**：按 **Agent**、**Provider**、**Model**、**Channel** 等过滤会话侧日志；网关数据源下可配合 **模块** 等条件收窄路由、HTTP 等子系统。
4. **日志级别与错误类**：级别多选；会话表在缺少标准 level 列时，可用 `message_is_error` 等方式近似区分 error 与非 error（见界面提示）；在支持的数据源上可 **仅看错误 / 排除错误**，快速聚焦异常。
5. **排序**：支持按时间、级别、子系统/模块、Agent 等排序，升序/降序可配。
6. **会话表字段级过滤（服务端）**：通过 `fieldFilters` 下发多条「字段键 + 取值」以 **AND** 组合进 SQL；单条可 **exclude** 排除取值；条件条数有上限（超出截断，见 `backend/log-search`）；**审计日志、网关数据源下**该能力不生效或会被清空，与表结构保持一致。

### 2.4 友好结果交互

1. **列显示控制**：按数据源展示不同列集，可通过表头或侧栏勾选控制列显隐，适配宽表与投屏场景。
2. **侧栏字段列表**：展示当前数据源相关字段；可展开查看 **当前页** 某字段的取值分布，辅助发现异常枚举值。
3. **行展开**：展开行可切换「字段网格」与「原始 JSON」，便于对照结构化视图与原始落库内容。
4. **分页与条数**：支持每页条数配置与上一页/下一页；总条数以服务端返回为准。
5. **时间分布图**：按时间分桶展示命中量；支持 **分桶粒度**（自动 / 分 / 时 / 日等）与 **点击柱形** 将查询时间范围收窄到该桶对应区间，便于从尖峰下钻。
6. **导出 / 下载**：支持将当前查询条件下的结果导出（如 CSV 等，具体格式以产品实现为准），便于离线分析或附件留存。

## 3. 应用场景

1. **线上故障排查**：告警触发后，运维在日志搜索中输入错误关键词或 Trace ID，叠加「最近 1 小时」与会话 ID，快速拉取 Agent 与网关相关日志，对比时间线定位是模型超时、工具失败还是下游拒绝连接。
2. **安全与合规审计**：安全岗选择「配置变更日志」数据源，结合操作人、资源关键词与「上月」时间窗，检索谁在何时修改了关键配置，并导出结果作为审计附件。
3. **客户问题复现与举证**：客服或技术支持拿到用户提供的 **Session ID** 或 **Request ID**，在会话日志中精确过滤，展开 JSON 查看模型回复与工具入参，对内复盘或对外说明问题原因。
4. **容量与异常模式分析**：研发选择「全部」或网关数据源，配合日志来源类别与级别过滤，观察某 Channel 或某模块在时间分布图上的尖峰，判断是否为活动洪峰、错误重试风暴或配置变更后的异常放量。
