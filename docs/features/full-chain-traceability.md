# 全链路追踪溯源功能说明

本功能可视为“会话深度追踪”模块中最尖锐、最垂直的一把下钻利刃。全链路追踪模块聚焦基于单一的调用标的（Trace/Span），采用符合 OpenTelemetry (OTel) 观测标准的呈现手法，将人工智能驱动工具流复杂的黑盒黑箱如同解码瀑布般完全剖开晾晒于白光之下。

---

## 一、模块定位

尽管有宏大日志及审计报表的存在，遇到顽固的偶发故障仍将面临深水区痛点：
- 执行阻塞无法定位定罪：系统耗时了几十秒才回复用户的请求，但没有人能在满屏杂碎代码报错日志里迅速说是大模型接口慢，还是数据库抽数锁闭超时。
- 调用逻辑穿插交织成谜：一次复合查询请求涉及三四个智能意图反转及内循环重试，只看最后一条输入和发还完全无济于事。
- 致命错误抛栈断层：业务崩溃时的案发现场上下文，经常被淹没冲刷找不出当时挂掉前携带了什么具体的传参变量与死机堆栈码。

依托于“执行任务的逐层剥离树 + 微距时间线瀑布延展 + 代码级的异常抛出留存快照”，打造系统级别的全穿透型排险显微镜。

---

## 二、功能实现说明

### 1) 逻辑意图拆分进程树 (Trace Breakdown Tree)
构建展示清晰父子执行归属从属关系的视觉图谱：
- **微观分化追踪**：以顶层的最初意图交互指令发端，次第展开启底下的所有中间环节，如大模型思考（Thoughts）、计划分发指令调度、实际内网跨应用的物理工具节点（Tool Calls）。
- **进程上下游依存串连**：使维护者马上能明了哪个被调起的外部爬虫动作或者本地 Shell 是由哪一句意图链派生的动作。

### 2) 花销与时间线横置铺陈瀑布流 (Waterfall Delay Visual)
诊断影响前端吞吐流转最大瓶颈所在的“X 光机”：
- **时间段切片透视**：把每个子步骤所花费的毫秒长条平铺展现。
- **致命慢查抓取定型**：哪一个接口通信请求卡停占用阻挡了全流进度，一瞥之间一目了然，杜绝互相瞎猜测推诿。

### 3) 报错报文深处快照提取与保留 (Error Context Catch)
- **参数报文对齐呈现**：对于有报警中断异常颜色的步骤色块（高频抛红点），点击能呈现瞬间系统的抓存状态、输入入参、崩溃返回的回包原始 JSON 及系统内抛异常栈文。

---

## 三、业务价值

### 1) 使排兵布阵级智能调优转为即战力现实
- 使原本难以调教、频生莫名歧途反馈的大模型 Prompt （系统提示词设定工程），拥有了像传统微服务后端排故调试级别的精密探针。

### 2) 大幅裁切无效网络耗时动作并降低风控延迟
- 有效识别由于代码机制及平台通信架构冗余拖慢了 Agent 处理响应速度导致超时崩溃的结构灾难。

---

## 四、典型应用场景

- **长尾请求极限性能拉包复盘**：被点名反馈一个极为卡顿的耗时会话过程，将涉事的 Trace ID 敲入并立刻查看到底是大模型反应慢了还是后续某个三方外挂组件在拖后腿。
- **致命越权或者死循环错误定点定案**：一个导致进程崩溃的错误触发后，深入找出它当时拿着什么违约格式的发参包去敲打系统的门致使引擎阻断，将其原貌提取补丁修复。
